# Contributing to LuciferCore Documentation

First off, thank you for considering contributing to the LuciferCore documentation! It's people like you who make LuciferCore a great tool for the .NET community.

## How Can I Contribute?

### Reporting Issues
* If you find a typo, a broken link, or an outdated code snippet, please open an **Issue** on GitHub.
* Describe the error and suggest a fix if possible.

### Improving Content
* You can help by clarifying existing guides or adding missing documentation for Public APIs.
* We appreciate contributions that make the documentation easier to understand for beginners while maintaining technical depth for advanced users.

### Translations (Localization)
* LuciferCore aims to support multiple languages (English and Vietnamese).
* If you are fluent in both, your help in refining translations is highly valued.

## Pull Request Process

1. **Fork** the repository and create your branch from `main`.
2. If you've added code examples, ensure they are tested and follow C# best practices.
3. Update the `SUMMARY.md` if you are adding new pages to ensure they appear in the GitBook sidebar.
4. Issue a **Pull Request** with a clear description of your changes.
5. Once your PR is merged, your changes will be automatically synced to our [GitBook site](https://bufmod.gitbook.io/lucifercore).

## Style Guidelines

* **Tone:** Professional, clear, and concise.
* **Format:** Use standard Markdown.
* **Terminology:** Keep technical terms consistent (e.g., use "Middleware" instead of translating it to a confusing local term).
* **Code:** Wrap all code snippets in appropriate C# syntax highlighting blocks.
